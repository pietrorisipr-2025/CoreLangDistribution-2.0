# CoreLangDistribution 2.0 alpha5 format

A `.cldrepo` is a directory repository, not a single archive.

```text
repo.cldrepo/
├── release.json
├── files.idx.json
├── chunks.idx.json
├── signatures.json        # optional, after signing
└── packs/
    └── pack-000000.cldpak
```

## release.json

Contains schema/version, `release_id`, monotonic `release_seq`, metadata `root_hash`, pack references, chunker configuration and storage metrics.

## files.idx.json

Array of files:

```text
path
size
file_hash
chunks: [{chunk_id, file_offset, raw_len}]
```

Paths are POSIX relative paths. Absolute paths and `..` are rejected.

## chunks.idx.json

Array of content-addressed chunks:

```text
chunk_id/raw_hash
compressed_hash
raw_len
compressed_len
codec
pack_id
pack_offset
pack_len
```

## signatures.json

Alpha5 signs a deterministic payload containing release id/seq, root hash, file index digest, chunk index digest and pack metadata.

Algorithm: `ed25519`.

The verifier uses a separate public trust key JSON generated by `keygen --pub-out`.

## patch plan

A `.cldpatch.json` binds an old release to a new release and lists only chunks that may be downloaded for the update. Alpha5 validates:

- target `release_id`, `release_seq`, `root_hash`;
- installed old `release_id`, `release_seq`, `root_hash`;
- every planned chunk’s hash, codec, offset and length against the target repo.

## Alpha8 update — trusted root and expiry

Alpha8 adds a local trusted-root policy file. The root is distributed out-of-band and contains one or more trusted Ed25519 public keys, optional root expiry and a minimal policy with `require_signature`, `min_release_seq` and release-expiry enforcement. Repositories may include `expires_at` and `not_before`; signatures cover these fields.

Relevant commands:

```bash
cld2 pack INPUT --out REPO --expires-at 2035-01-01T00:00:00Z
cld2 keygen --out release_private.json --pub-out release_public.json
cld2 sign REPO --key release_private.json
cld2 root-init --out trusted_root.json --key release_public.json --expires-at 2035-01-01T00:00:00Z --min-release-seq 1
cld2 verify REPO --deep --trusted-root trusted_root.json
cld2 fetch REPO --install INSTALL --trusted-root trusted_root.json
```
