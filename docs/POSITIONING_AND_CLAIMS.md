# Positioning and claims

## Correct positioning

CLD2 is best described as a **cost-aware distribution/update planner** for artifacts where chunk/object reuse matters.

Good target audiences:

- developers distributing large update artifacts;
- CI/CD and DevOps workflows;
- FinOps/cost-aware distribution reporting;
- object-store and cache-aware release systems;
- GitHub portfolio/vetrina with honest benchmarks.

## Strong claim

CLD2 is very strong when prior content exists and the update has high reuse.

## Weak/unsafe claim

Do not claim that CLD2 beats all tools in all cases. High entropy and heavy changes reduce or eliminate the advantage.

## Monetization

Prefer:

- open source or MIT-licensed release;
- GitHub Sponsors/donations;
- optional support;
- consulting/integration;
- managed packing/distribution;
- custom FinOps reports.

Avoid a mandatory paywall for first GitHub visibility.
